Cinderkeep - Asset Foldering List

Use this list when adding new files.

Core Folders
- Assets/Animations
  Animation clips and Animator Controllers.

- Assets/Arts
  Team-made models, sprites, textures, and source visual assets.

- Assets/Audio
  BGM, SFX, and audio-related assets.

- Assets/Data
  JSON, CSV, ScriptableObject, and static game data.

- Assets/Fonts
  Font files.

- Assets/Materials
  Shared Unity materials.

- Assets/Prefabs
  Reusable GameObjects.
  Use Player, Enemy, Building, Harvestables, Map, and UI subfolders first.

- Assets/Resources
  Only files loaded through Resources.Load.
  Do not use this as a general storage folder.

- Assets/Scenes
  Main_Lobby.unity is the main menu scene.
  MainGame/Cinderkeep_Game.unity is the actual game scene.

- Assets/Scripts
  Official C# scripts.

- Assets/Tests
  Official Unity Test Runner tests.
  Use EditMode and PlayMode subfolders.

- Assets/ThirdParty
  External assets and plugins.
  AssetStore/Paid is local-only and ignored by Git.
  AssetStore/Free is for shared free external assets.
  PrivateAssets is local-only and ignored by Git.
  Plugins is for shared plugin code that the team agrees to commit.

- Assets/UI
  UI-only sprites, images, and style assets.

Special Folders
- Assets/_Sandbox
  Temporary experiments, prototype scenes, prototype prefabs, and temporary scripts.
  Move useful work to official folders before merging into real gameplay.

- Assets/_Disabled
  Files removed from the active game but kept for reference.
  Disabled C# files must use .cs.disabled.

- Assets/_TeamGuide
  Team guide documents.

Scripts Folder
- Assets/Scripts/Managers
  GameManager, UIManager, SoundManager, GameDataManager, GameObjectManager, MapManager.

- Assets/Scripts/Data
  Static data classes such as EnemyData, ItemData, RecipeData.

- Assets/Scripts/Models
  Runtime save data such as PlayerModel, InventoryModel, GameRunModel.

- Assets/Scripts/Player
  Components attached to the Player object.

- Assets/Scripts/UI
  UI components such as HUDView, InventoryView, MainMenuController.

- Assets/Scripts/Systems
  Feature systems such as inventory, crafting, building, combat.

- Assets/Scripts/Interfaces
  Shared interfaces such as IDamageable, IInteractable, and IGameInitializable.

- Assets/Scripts/Utils
  Pure utility methods and helper functions.

- Assets/Scripts/Editor
  Editor-only tools.

Third Party Folders
- Assets/ThirdParty/AssetStore/Paid
  Paid Asset Store files. This folder is ignored by Git.

- Assets/ThirdParty/AssetStore/Free
  Free third-party files that the team can commit when the license allows it.

- Assets/ThirdParty/PrivateAssets
  Team-private files shared outside Git. This folder is ignored by Git.

- Assets/ThirdParty/Plugins
  External plugins that are safe to share in Git.

Recovery Folder
- Assets/_Disabled/Recovery
  Unity recovery scenes or temporary rescued files. These are not active project files.
