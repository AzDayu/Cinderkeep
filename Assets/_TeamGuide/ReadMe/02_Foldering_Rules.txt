폴더링 규칙

목적
- 팀원이 새 파일을 만들 때 어디에 넣어야 하는지 바로 알 수 있게 합니다.
- 임시 실험, 정식 기능, 삭제 후보가 섞이지 않게 합니다.
- 기존 스크립트는 삭제하지 않고 역할에 맞는 폴더로 이동합니다.

정식 프로젝트 폴더
- Assets/Animations
  애니메이션 클립, Animator Controller를 둡니다.

- Assets/Arts
  직접 만든 3D 모델, 2D 이미지, 텍스처 같은 시각 에셋을 둡니다.

- Assets/Audio
  BGM, SFX, AudioMixer를 둡니다.

- Assets/Data
  JSON, CSV, ScriptableObject 같은 기획 데이터를 둡니다.

- Assets/Materials
  공용 Material을 둡니다.

- Assets/Prefabs
  재사용할 GameObject를 둡니다.
  Player, Enemy, Building, Harvestables, Map, UI 하위 폴더를 우선 사용합니다.

- Assets/Resources
  코드에서 Resources.Load로 직접 불러야 하는 파일만 둡니다.
  아무 에셋이나 넣는 보관소로 쓰지 않습니다.

- Assets/Scenes
  실제 Unity 씬을 둡니다.
  Main_Lobby.unity는 메인 메뉴, MainGame/Cinderkeep_Game.unity는 실제 게임 씬입니다.

- Assets/Scripts
  정식 C# 스크립트를 둡니다.
  Managers, Data, Models, Player, UI, Systems, Interfaces, Utils, Editor 하위 폴더를 사용합니다.

- Assets/Tests
  Unity Test Runner용 정식 테스트 코드를 둡니다.
  EditMode와 PlayMode를 나누어 관리합니다.

- Assets/ThirdParty
  외부 에셋, 에셋스토어 패키지, 외부 플러그인을 둡니다.
  AssetStore/Paid에는 유료 에셋을 둡니다. 이 폴더 내용은 Git에 올리지 않습니다.
  AssetStore/Free에는 팀이 공유할 수 있는 무료 외부 에셋을 둡니다.
  PrivateAssets에는 개인이 따로 받은 에셋을 둡니다. 이 폴더 내용은 Git에 올리지 않습니다.
  Plugins에는 팀이 공유하기로 한 플러그인을 둡니다.

- Assets/UI
  UI 전용 이미지, 스프라이트, 폰트 스타일, UI 관련 에셋을 둡니다.

특수 폴더
- Assets/_Sandbox
  실험 씬, 임시 프리팹 조립, 임시 스크립트를 둡니다.
  정식 기능으로 승격되면 반드시 Scripts, Prefabs, Scenes 등 역할 폴더로 옮깁니다.

- Assets/_Disabled
  당장 삭제는 어렵지만 현재 게임에서 사용하지 않는 파일을 둡니다.
  C# 스크립트는 .cs.disabled처럼 확장자를 바꿔 컴파일되지 않게 합니다.

- Assets/_TeamGuide
  팀 규칙, 폴더링, Git 사용법, 코드 스타일 안내 문서를 둡니다.

Scripts 하위 규칙
- Assets/Scripts/Managers
  GameManager, UIManager, SoundManager, GameDataManager, GameObjectManager처럼 전체 흐름이나 공통 관리를 맡는 스크립트를 둡니다.

- Assets/Scripts/Data
  EnemyData, ItemData, RecipeData처럼 변하지 않는 기획 데이터 구조를 둡니다.

- Assets/Scripts/Models
  PlayerModel, InventoryModel처럼 플레이 중 변하고 저장이 필요한 데이터를 둡니다.

- Assets/Scripts/Player
  Player 오브젝트에 직접 붙는 이동, 점프, 시점, 상태 스크립트를 둡니다.

- Assets/Scripts/UI
  HUD, 인벤토리, 메인 메뉴처럼 UI 오브젝트에 붙는 스크립트를 둡니다.

- Assets/Scripts/Systems
  제작, 건축, 인벤토리처럼 기능 단위로 묶이는 시스템 스크립트를 둡니다.

- Assets/Scripts/Interfaces
  IDamageable, IInteractable처럼 여러 스크립트가 함께 쓰는 인터페이스를 둡니다.

- Assets/Scripts/Utils
  순수 계산, 변환, 공용 보조 기능을 둡니다.

- Assets/Scripts/Editor
  씬 자동 생성, 테스트맵 생성처럼 Unity Editor에서만 쓰는 도구를 둡니다.

주의
- main에 직접 작업하지 않습니다.
- 새 폴더를 만들기 전에 기존 역할 폴더에 들어갈 수 있는지 먼저 봅니다.
- 스크립트 1~2개짜리 작은 기능은 억지로 전용 폴더를 만들지 않습니다.
- 같은 파일을 여러 폴더에 복사하지 않습니다.
- _Sandbox 안의 코드는 정식 기능이 아니므로, 완성되면 반드시 역할 폴더로 승격합니다.
