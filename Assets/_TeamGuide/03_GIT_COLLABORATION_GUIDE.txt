Cinderkeep - Git Collaboration Guide

Branch Rule
- Create every new task branch from the latest main.
- Do not work directly on main.
- Keep one branch focused on one task.

Branch Name Examples
- feature/player-movement
- feature/building-basic
- feature/resource-gathering
- feature/enemy-ai
- fix/main-menu-start

Scene Ownership Rule
- Avoid multiple people editing the same .unity scene at the same time.
- Cinderkeep_Game is the main game scene, so scene changes need extra care.
- Prefer prefabs and scripts when possible.
- If two people must edit the same scene, decide who owns the scene edit first.

Folder Rule
- Put new scripts in the correct Assets/Scripts role folder.
- Use Assets/_Sandbox for experiments.
- Move useful _Sandbox work into official folders before merge.
- Put inactive files in Assets/_Disabled.

Prefab Rule
- Reusable objects should become prefabs.
- Put gameplay prefabs under Assets/Prefabs by role.
- Put temporary prefab experiments under Assets/_Sandbox/Prefabs.

Commit Rule
- Commit small, understandable changes.
- Good commit scope:
  Add player movement component.
  Add basic building prefab.
  Update resource node interaction.
- Bad commit scope:
  Massive random changes.
  Unrelated scene and script edits together.

Branch Absorption Rule
- 팀원 브랜치는 그대로 덮어쓰지 않습니다.
- 먼저 작업 의도를 확인하고, 현재 main 구조에 맞게 필요한 부분만 흡수합니다.
- 큰 main 업데이트 이전에 만들어진 브랜치는 좋은 아이디어만 살리고, 오래된 씬/스크립트 상태를 되살리지 않습니다.
- main 씬 오브젝트, 매니저 연결, HUD 배치, ProjectSettings는 의도가 분명할 때만 변경합니다.

Absorption Checklist
- 살릴 의도:
  어떤 게임플레이, 프리팹, 데이터, 모델, UI, 스크립트 책임을 살릴지 적습니다.
- 제외할 코드 또는 파일:
  오래된 씬 변경, 중복 스크립트, 테스트 파일, 깨진 에셋, 관련 없는 수정은 제외합니다.
- main에 맞춰 수정한 파일:
  폴더링, 네이밍, 코드 스타일에 맞추기 위해 수정한 파일을 적습니다.
- 다음 작업 기준 파일:
  팀원이 다음 작업을 이어갈 때 기준으로 봐야 할 파일을 적습니다.
- 커밋 설명:
  살린 것, 수정한 것, 일부러 제외한 것을 커밋 설명에 남깁니다.

Before Pull Request Or Merge
- Pull latest main.
- Open Unity and let assets import.
- Check Console errors.
- Test the scene or prefab you changed.
- Check that no unrelated files were changed.
- Mention which folder, scene, and system you touched.

Merge Conflict Warning
- Unity YAML scene conflicts are hard to resolve by guessing.
- Do not auto-resolve scene conflicts without checking in Unity.
- If a scene conflict happens, ask the team before choosing one side.
