Cinderkeep - Read First

Purpose
- This project is the shared Cinderkeep game project.
- The old lobby convenience scenes are no longer part of the active project flow.
- Main_Lobby is the game main menu.
- Cinderkeep_Game is the actual game development scene.

Main Scenes
- Main_Lobby: main menu scene. Play Mode starts here.
- MainGame/Cinderkeep_Game: actual game scene. Open this scene when editing gameplay.

Editor Flow
- When Unity opens, the editor should show Cinderkeep_Game for direct game work.
- When Play is pressed, the project starts from Main_Lobby.
- This keeps editing focused on the real game while preserving the player-facing menu flow.

Team Rules
- Work on a personal branch, not directly on main.
- Put new files in the correct role folder.
- Use _Sandbox for experiments.
- Move useful _Sandbox work into official folders before merging.
- Put removed or inactive files in _Disabled.
- Do not leave active C# files in _Disabled. Use .cs.disabled.
- Keep components small and named by role.
- Do not collect unrelated references in one controller.
- Check changed files before commit and push.

Important Folders
- Assets/Scripts/Managers: GameManager, UIManager, SoundManager, GameDataManager, GameObjectManager.
- Assets/Scripts/Player: Player object components.
- Assets/Scripts/UI: UI object components.
- Assets/Scripts/Systems: feature systems such as crafting, building, combat, inventory.
- Assets/Tests: official Unity Test Runner tests.
- Assets/_Sandbox: prototype and experiment work.
- Assets/_Disabled: inactive files kept for reference.
- Assets/_TeamGuide: team guide documents.
