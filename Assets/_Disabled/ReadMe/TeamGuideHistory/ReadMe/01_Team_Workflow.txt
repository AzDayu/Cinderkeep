팀 작업 흐름

1. 최신 main을 pull합니다.
2. 개인 브랜치를 만듭니다.
3. Cinderkeep_Game 씬 또는 담당 프리팹에서 작업합니다.
4. 실험이 필요하면 Assets/_Sandbox에서 먼저 만듭니다.
5. 정식 기능으로 쓰기로 결정되면 역할 폴더로 옮깁니다.
6. Unity Console 오류를 확인합니다.
7. 변경 파일을 확인하고 커밋합니다.
8. push 후 팀장에게 머지 검토를 요청합니다.

권장 작업 위치
- 플레이어: Assets/Scripts/Player
- UI: Assets/Scripts/UI
- 제작, 건축, 전투, 인벤토리: Assets/Scripts/Systems
- 데이터 구조: Assets/Scripts/Data
- 저장/진행 데이터: Assets/Scripts/Models
- 실험 작업: Assets/_Sandbox
- 비활성 보관: Assets/_Disabled
