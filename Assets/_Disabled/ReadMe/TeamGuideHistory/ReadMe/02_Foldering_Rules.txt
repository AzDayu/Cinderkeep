Cinderkeep 폴더링 규칙

목적
- 팀원이 새 파일을 만들 때 어느 폴더에 넣어야 하는지 바로 알 수 있게 합니다.
- 실험 파일, 정식 기능, 삭제 후보가 섞이지 않게 합니다.
- 기능을 지우지 않고 역할에 맞는 폴더로 옮기는 것을 원칙으로 합니다.

정식 프로젝트 폴더
- Assets/Animations
  Animation Clip, Animator Controller를 둡니다.

- Assets/Arts
  팀이 직접 쓰는 3D 모델, 2D 이미지, 텍스처 원본을 둡니다.

- Assets/Audio
  BGM, SFX, AudioMixer를 둡니다.

- Assets/Data
  JSON, CSV, ScriptableObject 같은 기획 데이터를 둡니다.

- Assets/Fonts
  폰트 파일과 TMP Font Asset을 둡니다.

- Assets/Materials
  공용 Material을 둡니다.

- Assets/Prefabs
  재사용할 GameObject를 둡니다.
  Player, Enemy, Building, Harvestables, Map, UI 하위 폴더를 우선 사용합니다.

- Assets/Resources
  코드에서 Resources.Load로 직접 불러와야 하는 파일만 둡니다.
  일반 보관 폴더로 쓰지 않습니다.

- Assets/Scenes
  Unity 씬 파일을 둡니다.
  Main_Lobby.unity는 메인 메뉴, MainGame/Cinderkeep_Game.unity는 실제 게임 씬입니다.

- Assets/Scripts
  정식 C# 스크립트를 둡니다.
  역할별 하위 폴더를 우선 사용합니다.

- Assets/Settings
  URP, Input System 같은 프로젝트 설정 에셋을 둡니다.

- Assets/Tests
  Unity Test Runner용 정식 테스트 코드를 둡니다.
  EditMode와 PlayMode를 나눠서 관리합니다.

- Assets/ThirdParty
  외부 에셋과 플러그인을 둡니다.
  AssetStore/Free는 커밋 가능한 무료 외부 에셋입니다.
  AssetStore/Paid와 PrivateAssets는 Git에 올리지 않습니다.
  Plugins는 팀이 공유하기로 정한 플러그인만 둡니다.

- Assets/UI
  UI 전용 이미지, 스프라이트, 스타일 에셋을 둡니다.

특수 폴더
- Assets/_Sandbox
  실험 씬, 임시 프리팹, 임시 스크립트를 둡니다.
  정식 기능으로 승격되면 반드시 정식 폴더로 옮깁니다.

- Assets/_Disabled
  현재 게임에서 빠졌지만 바로 삭제하지 않을 파일을 둡니다.
  C# 파일은 .cs.disabled 확장자를 사용해서 컴파일되지 않게 합니다.

- Assets/_TeamGuide
  팀 규칙, 폴더링, Git 사용법, 코드 스타일 안내 문서를 둡니다.

Scripts 하위 폴더 규칙
- Assets/Scripts/Managers
  GameManager, UIManager, SoundManager, GameDataManager, GameObjectManager처럼 전체 흐름이나 공통 관리를 맡는 스크립트를 둡니다.

- Assets/Scripts/Data
  EnemyData, ItemData, RecipeData처럼 변하지 않는 기획 데이터 구조를 둡니다.

- Assets/Scripts/Models
  PlayerModel, GameRunModel처럼 플레이 중 변하고 저장이 필요한 데이터를 둡니다.

- Assets/Scripts/Player
  Player 오브젝트에 직접 붙는 이동, 점프, 시점, 상태, 공격, 상호작용 스크립트를 둡니다.

- Assets/Scripts/Enemy
  몬스터 감지, 이동, 공격, 체력, HUD 스크립트를 둡니다.

- Assets/Scripts/UI
  HUD, 메인 메뉴, 인벤토리, 게임오버 UI 스크립트를 둡니다.

- Assets/Scripts/Systems
  자원, 건축, 전투, 입력 래퍼처럼 기능 단위 시스템을 둡니다.
  스크립트가 3개 이상 유기적으로 묶이면 하위 폴더를 만듭니다.

- Assets/Scripts/Interfaces
  IInteractable, IGameInitializable처럼 여러 스크립트가 함께 쓰는 인터페이스를 둡니다.

- Assets/Scripts/Utils
  순수 계산, 변환, 공용 보조 기능을 둡니다.

- Assets/Scripts/Editor
  Unity Editor에서만 실행되는 씬 빌더, Bake 도구, 검증 도구를 둡니다.

주의
- main에 직접 작업하지 않습니다.
- 새 폴더를 만들기 전에 기존 역할 폴더에 들어갈 수 있는지 먼저 확인합니다.
- 스크립트 1~2개짜리 작은 기능은 불필요한 전용 폴더를 만들지 않습니다.
- 같은 파일을 여러 폴더에 복사하지 않습니다.
- _Sandbox 안의 코드는 정식 기능이 아니므로 완성되면 반드시 역할 폴더로 승격합니다.
