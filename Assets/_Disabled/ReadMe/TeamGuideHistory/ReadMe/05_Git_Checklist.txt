Git 체크리스트

커밋 전 확인
- Unity Console 에러가 없는가?
- 내가 바꾼 씬이 의도한 씬인가?
- 다른 사람 작업 파일을 실수로 바꾸지 않았는가?
- Play 버튼으로 최소 1번 실행 확인했는가?
- 변경한 기능을 한 문장으로 설명할 수 있는가?

커밋 메시지 규칙
- 제목은 한글로 짧고 명확하게 씁니다.
- 설명은 핵심 변경점만 적습니다.

예시
제목: 메인 로비 ReadMe 버튼 추가
설명: ReadMe 폴더를 열 수 있는 버튼과 팀 안내 문서를 추가함

머지 전 확인
- 팀장 또는 담당자가 실제 실행을 확인합니다.
- 충돌이 나면 임의로 지우지 말고 어떤 파일 충돌인지 먼저 공유합니다.
