팀원 공유용 문서입니다.

폴더 생성 기준

1. 스크립트가 1~2개뿐인 작은 기능이면 전용 하위 폴더를 만들지 말고 기존 폴더 바로 아래에 둡니다.
   예: 간단한 점수 계산, 간단한 낮밤 전환, 단일 보조 컴포넌트

2. 스크립트가 3개 이상이고 서로 유기적으로 얽혀 있으면 전용 폴더를 만들어 격리합니다.
   예: Combat, Inventory, Crafting, Building, Enemy, Player

3. Controller는 장면 순서 지휘와 큐 관리만 담당합니다.
   많은 참조를 Controller 하나에 몰지 말고, 각 오브젝트가 자기 역할 컴포넌트를 직접 가지게 합니다.

4. UI는 런타임에서 new GameObject로 만들지 않습니다.
   미리 만든 씬 오브젝트나 Prefab을 연결해서 사용합니다.

5. Resources 폴더는 일반 보관소가 아닙니다.
   코드에서 Resources.Load가 필요한 파일만 넣습니다.

6. 외부 에셋은 Assets/ThirdParty 아래에만 넣습니다.
   유료 에셋과 개인 공유 에셋은 Git에 올리지 않습니다.

7. _Sandbox에 있는 파일은 임시 상태입니다.
   실제 게임에 연결하기 전에 공식 폴더로 승격하거나, 폐기할 파일은 _Disabled로 옮깁니다.
