Cinderkeep - Cinderkeep Shared Workspace Plan

Planning Deck Summary
- Cinderkeep is inspired by Muck's fast growth feeling.
- The direction is single-player base-defense roguelite.
- Day is for farming and preparation.
- Night is for survival and base defense.
- Chests create random growth and replay variety.
- Better resources should be farther away and riskier.
- The build should prove a fun short session first.

Suggested Core Systems
Implement these in Cinderkeep_Game, not directly in the workspace scene.

- First-person movement and camera.
- Wheel zoom in/out.
- Day/night phase timer.
- Resource gathering.
- Basic crafting.
- Basic building or wall placement.
- Player health, hunger, and stamina HUD.
- Base core health.
- Night enemy wave.
- Random chest upgrade.
- Restart loop.

Suggested Team Split
- Planning/PM: feature priority, task board, decisions.
- Player/HUD: movement, zoom, health, hunger, stamina.
- Gathering/Crafting/Building: resources, tools, tier upgrades, structures.
- Combat/AI/Boss: wave enemies, player chaser, base breaker, first boss.
- Map/Balance/QA: resource distance, respawn, day 3 test loop.

Important Scope Rule
- Do not build everything at once.
- First prove:
  player can move,
  gather a resource,
  open a chest,
  defend a base core at night,
  restart after failure.
