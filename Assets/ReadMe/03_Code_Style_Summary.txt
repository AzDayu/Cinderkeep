코드 스타일 요약

기준
- 강사님의 코드일관화용 AI PDF 스타일을 기준으로 유지합니다.

핵심 원칙
- Controller는 장면 흐름과 큐 관리만 담당합니다.
- 오브젝트는 자기 역할 컴포넌트를 직접 가집니다.
- UI는 가능한 프리팹이나 미리 만든 오브젝트를 사용합니다.
- 한 Controller에 참조가 과하게 몰리면 컴포넌트로 분리합니다.
- 공통 기능은 Manager와 데이터 기반 구조로 관리합니다.
- 멤버 변수는 _prefix를 사용합니다.
- Unity 참조는 [SerializeField] private로 둡니다.
- 외부 접근이 필요하면 프로퍼티를 사용합니다.
- 메서드 이름은 동사로 시작합니다.

예시
- PlayerMovement: 이동만 담당
- PlayerInteraction: 상호작용만 담당
- WorkspaceActor: 작업 공간 오브젝트 정보만 담당
- SceneLoadButton: 씬 이동 버튼만 담당
- ReadMeFolderOpenButton: ReadMe 폴더 열기만 담당
