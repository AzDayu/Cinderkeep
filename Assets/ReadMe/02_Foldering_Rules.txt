폴더링 규칙

공용 에셋
- Assets/MainAssets
- 여러 명이 같이 쓰는 이미지, 사운드, 프리팹, 데이터, 문서를 둡니다.

개인 에셋
- Assets/PersonalAssets/캐릭터_이름
- 개인 실험용 스크립트, 이미지, 사운드, 프리팹을 둡니다.

읽기 문서
- Assets/ReadMe
- 팀 규칙, 작업 흐름, 코드 스타일 요약, Git 체크리스트를 둡니다.

주의
- 공용 폴더 파일을 바꾸기 전에는 목적을 분명히 합니다.
- 같은 이름의 파일을 여러 폴더에 복사하지 않습니다.
- 테스트용 파일은 이름에 Test 또는 Prototype을 붙여 구분합니다.
